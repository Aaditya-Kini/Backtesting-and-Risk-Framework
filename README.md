# Systematic Multi-Asset Backtesting & Risk Framework

## Executive Summary
I engineered this institutional-grade, strictly event-driven backtesting engine to rigorously evaluate quantitative trading strategies across equities, fixed income, and commodities. 

Most open-source or academic backtesters rely on Pandas vectorization, which frequently suffers from look-ahead bias and assumes perfect execution at closing prices. My goal was to build a framework that simulates the actual constraints of a live trading desk. By utilizing an asynchronous event queue, this engine explicitly models market microstructure, including bid-ask spread crossing, variable slippage, commission drag, and dynamic portfolio optimization.

## System Architecture & The Event Loop
To guarantee zero data leakage, I designed the framework around an isolated, sequential event loop. The system processes market data tick-by-tick (or day-by-day) using four core event classes:

1. **`MarketEvent`**: Fetches new pricing/volume data and updates the central state.
2. **`SignalEvent`**: Generated by the Alpha modules. Contains the asset ticker, directional bias (Long/Short), and signal conviction.
3. **`OrderEvent`**: Generated by the `Portfolio` module after intercepting signals, calculating risk-adjusted position sizes, and checking available margin.
4. **`FillEvent`**: Returned by the `ExecutionHandler` after simulating the exchange routing. It applies explicit transaction costs and a volume-based slippage penalty before updating portfolio holdings.

Because the system is fully modular, moving from historical CSV data to live WebSocket trading simply requires swapping out the `DataHandler` class; the underlying Alpha and Risk logic remains entirely untouched.

## Alpha Generation Models

I implemented several uncorrelated strategy modules to test the framework's versatility across different asset classes.

### 1. Advanced Cross-Sectional Momentum (Equities)
Rather than building a basic time-series momentum model, I implemented a cross-sectional relative strength strategy across a universe of liquid ETFs. 
* **The Logic:** The model ranks assets based on historical lookback returns and dynamically allocates capital to the top outperforming decile while shorting the bottom decile to maintain dollar-neutrality.
* **Risk Parity Sizing:** To prevent highly volatile assets from dominating the portfolio variance, I implemented **Inverse Volatility Weighting**. Capital is distributed such that each asset contributes an equal amount of risk:
  $$w_{i,t} = \frac{\sigma_{i,t}^{-1}}{\sum_{j \in U_{L,t}} \sigma_{j,t}^{-1}}$$
* **Turnover Mitigation:** Cross-sectional strategies are highly susceptible to commission drag due to constant re-ranking. I engineered a "rebalancing buffer" (hysteresis banding) that only liquidates a position if its rank drops a specific percentage below the target threshold, significantly reducing unnecessary churn.

### 2. Yield Curve Statistical Arbitrage (Fixed Income)
To handle rates and macro trading, I built a fixed-income module that tracks the sovereign yield curve.
* **The Logic:** Utilizes a Nelson-Siegel curve fitting algorithm to identify structural mispricings (e.g., in the 2Y-10Y spread). 
* **Risk Management:** The strategy computes portfolio Modified Duration and Convexity to ensure all trades are executed as strictly **duration-neutral** spread trades, isolating the alpha of the curve shape from directional interest rate risk.

## Dynamic Portfolio Optimization & Risk Management
Capital allocation is handled dynamically via a central `Portfolio` module. I integrated `scipy.optimize` to apply **Mean-Variance Optimization (MVO)** across all active signals. 

At each rebalancing period, the engine computes the historical covariance matrix of the active assets and solves for the weight vector ($w$) that maximizes the Sharpe Ratio, subject to strict constraints capping the Maximum Drawdown and bounding the 95% Value at Risk (VaR).

## Statistical Validation of Alpha
To prove that the generated returns are driven by strategy edge and not generalized market beta, I run an automated Ordinary Least Squares (OLS) regression of the strategy's daily returns against the **Fama-French Three-Factor Model**:

$$R_{p,t} - R_{f,t} = \alpha + \beta_1(MKT_t) + \beta_2(SMB_t) + \beta_3(HML_t) + \epsilon_t$$

By isolating the $p$-value of the intercept ($\alpha$), the framework programmatically verifies whether the strategy possesses statistically significant edge independent of market ($MKT$), size ($SMB$), and value ($HML$) factors.
